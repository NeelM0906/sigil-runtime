## API Patterns

### SELECT
```python
GET /rest/v1/{table}?column=eq.value&select=*&limit=100&order=created_at.desc
```

### INSERT
```python
POST /rest/v1/{table}
Body: JSON object
Header: Prefer: return=representation
```

### UPDATE
```python
PATCH /rest/v1/{table}?id=eq.{uuid}
Body: JSON with fields to update
```

### UPSERT (insert or update)
```python
POST /rest/v1/{table}?on_conflict=id
Header: Prefer: resolution=merge-duplicates,return=representation
```

### DELETE
```python
DELETE /rest/v1/{table}?id=eq.{uuid}
```

### COUNT
```python
GET /rest/v1/{table}?select=id
Header: Prefer: count=exact, Range: 0-0
Response header: Content-Range: 0-0/487
```

## Filter Operators (PostgREST)
| Operator | Meaning | Example |
|----------|---------|---------|
| `eq` | equals | `?status=eq.open` |
| `neq` | not equals | `?status=neq.closed` |
| `gt` | greater than | `?balance=gt.0` |
| `lt` | less than | `?importance=lt.5` |
| `gte` | >= | `?created_at=gte.2026-01-01` |
| `lte` | <= | `?created_at=lte.2026-12-31` |
| `like` | pattern | `?carrier=like.*Aetna*` |
| `ilike` | case-insensitive | `?name=ilike.*smith*` |
| `in` | in list | `?status=in.(open,pending)` |
| `is` | null check | `?notes=is.null` |

## Important Notes
- JSON fields (metadata, key_outcomes): always `json.dumps()` before inserting
- Timestamps: Supabase auto-sets `created_at` — don't set manually unless needed
- UUIDs: auto-generated by `gen_random_uuid()` — don't set manually
- SQL migrations can ONLY be run via the Supabase dashboard SQL editor (REST API doesn't expose raw SQL)
- Max response size: use `limit` param, paginate with `offset` for large datasets
